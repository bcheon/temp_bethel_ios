# 03. 시퀀스 다이어그램

이 문서는 BKC 푸쉬 시스템의 5가지 핵심 플로우를 단계별로 보여줍니다. 다이어그램이 안 보이면 [`README.md`의 다이어그램 안 보일 때](README.md#다이어그램-안-보일-때) 섹션 참고.

---

## 1. 앱 첫 실행 + 그룹 온보딩 + 구독 등록

신규 사용자가 앱을 처음 열었을 때.

```mermaid
sequenceDiagram
    autonumber
    participant U as 사용자
    participant App as BKCApp
    participant AD as AppDelegate
    participant FB as FirebaseApp
    participant FCM as FCM SDK
    participant View as GroupOnboardingView
    participant GS as GroupStore
    participant PS as PushService
    participant API as BKCAPIClient
    participant WP as WordPress<br/>/wp-json/bkc/v1
    participant FCMS as FCM 서버

    U->>App: 앱 실행
    App->>FB: FirebaseApp.configure()
    App->>AD: didFinishLaunchingWithOptions
    AD->>FCM: registerForRemoteNotifications()
    AD->>U: 알림 권한 요청 (UNUserNotificationCenter)
    U-->>AD: 허용
    AD->>FCM: APNs 토큰 획득
    FCM-->>AD: didReceiveRegistrationToken (FCM 토큰)
    AD->>PS: handleTokenRefresh(token)
    Note over PS: hasOnboarded == false 이므로<br/>토큰만 보관, 서버 등록은 onboarding 후

    App->>View: GroupOnboardingView 렌더
    View->>U: "전체"(필수) + "청년부"(선택) + "새가족"(선택) 표시
    U-->>View: 청년부 체크 → "계속" 탭
    View->>GS: completeOnboarding(["youth"])
    GS->>GS: 자동으로 'all' 추가 → ["all", "youth"]
    GS->>PS: subscribe(toGroup: "all")
    PS->>FCMS: subscribe to topic 'bkc_all'
    FCMS-->>PS: ok
    GS->>PS: subscribe(toGroup: "youth")
    PS->>FCMS: subscribe to topic 'bkc_youth'
    FCMS-->>PS: ok
    GS->>API: subscribe(token, deviceID, ["all","youth"])
    API->>WP: POST /subscribe {fcm_token, device_id, groups}
    WP->>WP: rate-limit 체크 (10/min/IP)
    WP->>WP: token / device_id / groups 검증
    WP->>WP: BKC_Subscriptions::upsert(...)
    WP-->>API: 200 {status:"ok", groups:[...]}
    API-->>GS: 성공
    GS->>GS: UserDefaults에 groups 저장 + hasOnboarded=true
    GS-->>View: 완료
    View->>App: dismiss → RootTabView 표시
```

**실패 시 롤백:** 위 5번~10번 중간에 실패하면 `GroupSync.apply`가 이미 성공한 subscribe를 unsubscribe로 되돌립니다 ([IRON RULE #5](../doc/testing.md)).

## 2. 관리자가 새 공지를 발송한다

WP 관리자 페이지에서 발송 버튼을 누른 순간부터 첫 디바이스에 도달하기까지.

```mermaid
sequenceDiagram
    autonumber
    participant Admin as 관리자
    participant Browser as 브라우저
    participant WP as WP 어드민<br/>(BKC_Admin)
    participant Camp as BKC_Campaigns
    participant DB as MySQL
    participant AS as Action Scheduler
    participant Disp as BKC_Dispatcher
    participant Subs as BKC_Subscriptions
    participant FCMC as BKC_FCM_Client
    participant Cache as WP Transient
    participant OAuth as oauth2.googleapis.com
    participant FCM as fcm.googleapis.com

    Admin->>Browser: "새 공지 작성" 메뉴 클릭
    Browser->>WP: GET /wp-admin/admin.php?page=bkc-push-compose
    WP->>WP: idempotency_token = UUID v4 생성
    WP-->>Browser: 폼 + hidden input(idempotency_token) + nonce

    Admin->>Browser: 제목/본문/그룹/딥링크 입력 → 제출
    Browser->>WP: POST 폼
    WP->>WP: nonce 검증 + manage_options 권한 체크
    WP->>Camp: find_by_uuid(idempotency_token)
    Camp->>DB: SELECT
    DB-->>Camp: NULL (첫 제출이므로)
    WP->>DB: INSERT wp_bkc_campaigns (status='queued', uuid=token)
    WP->>AS: as_schedule_single_action(time+10, 'bkc_dispatch_campaign', uuid)
    WP-->>Browser: "발송 큐에 등록되었습니다. UUID: ..."

    Note over AS: 10초 경과
    AS->>Disp: dispatch_handler({uuid: ...})
    Disp->>Camp: find_by_uuid → status == 'queued' 확인
    Disp->>Camp: transition_status(queued → sending)<br/>WHERE status='queued' 원자성
    Camp->>DB: UPDATE → 1 row affected
    Disp->>Subs: count_targeted(['all','youth'])
    Subs->>DB: SELECT COUNT(*) WHERE last_seen > now-14d AND JSON_CONTAINS(groups,...)
    DB-->>Subs: 423
    Disp->>DB: UPSERT wp_bkc_campaign_stats (subscribers_targeted=423)

    Disp->>FCMC: send_to_groups(title, body, deep_link, ['all','youth'], uuid)
    Note over FCMC: 'all' 포함이므로<br/>topic='bkc_all' 단일 사용
    FCMC->>Cache: get_transient('bkc_fcm_access_token_...')
    alt 캐시 미스
        FCMC->>FCMC: build_jwt() RS256 서명
        FCMC->>OAuth: POST /token (JWT bearer)
        OAuth-->>FCMC: {access_token, expires_in: 3600}
        FCMC->>Cache: set_transient(token, 50min TTL)
    end
    FCMC->>FCM: POST /v1/projects/.../messages:send<br/>{message:{topic:'bkc_all', notification, data:{campaign_uuid,deep_link}, apns:{mutable-content:1}}}
    FCM-->>FCMC: 200 {name: "projects/.../messages/0:..."}
    FCMC-->>Disp: {message_id, error: null}
    Disp->>Camp: transition_status(sending → sent, sent_at=NOW(), fcm_message_ids)
    Camp->>DB: UPDATE
```

**핵심 IRON RULE:** 같은 idempotency_token으로 두 번 제출해도 캠페인은 1개만 만들어집니다 (`Test_Idempotency.php`).

## 3. iPhone이 푸쉬를 받는다 (앱 백그라운드/종료 상태)

FCM이 발송한 후 사용자 디바이스에서 일어나는 일.

```mermaid
sequenceDiagram
    autonumber
    participant FCM as FCM
    participant APNs as Apple APNs
    participant iOS as iOS 시스템
    participant NSE as NotificationServiceExtension
    participant Banner as 알림 배너
    participant U as 사용자
    participant App as BKC App
    participant ANC as UNUserNotificationCenter
    participant ADel as AppDelegate
    participant TS as TelemetryService
    participant DLR as DeepLinkRouter
    participant WP as WordPress /events

    FCM->>APNs: APNs payload (mutable-content:1)
    APNs->>iOS: push 도착
    iOS->>NSE: didReceive(request, contentHandler)
    NSE->>NSE: campaign_uuid 추출
    NSE->>NSE: App Group UserDefaults에서 device_id 읽기
    NSE->>WP: POST /events {events:[{campaign_uuid, device_id, event_type:'delivered', occurred_at}]}
    Note over NSE,WP: 5초 budget · 실패 시 1회 재시도
    WP-->>NSE: 200 {accepted: 1}
    NSE->>iOS: contentHandler(bestAttemptContent)
    iOS->>Banner: 시스템 알림 배너 표시

    U->>Banner: 탭
    Banner->>iOS: launch app + UNNotificationResponse
    iOS->>App: didFinishLaunching (또는 백그라운드 wake)
    iOS->>ADel: didReceive(response, completionHandler)
    ADel->>TS: logOpened(campaignUUID)
    TS->>TS: pending 버퍼에 append (UserDefaults)
    alt response.userInfo에 deep_link 존재
        ADel->>DLR: route(url: deepLink)
        DLR->>DLR: DeepLink.from(url) → .news / .sermon / .event / .external
        DLR->>App: selectedTab 전환 + injectedHomeURL 설정
        ADel->>TS: logDeeplinked(campaignUUID)
    end
    ADel->>iOS: completionHandler()

    Note over TS: 5분 주기 또는 다음 fcm 토큰 변경 시
    TS->>WP: POST /events 배치 (최대 100건)
    WP-->>TS: 200 {accepted: N}
    TS->>TS: pending 버퍼에서 N건 제거
```

**대안 시나리오 — 앱이 포그라운드에 있을 때:** `willPresent`가 호출되어 `delivered` 이벤트가 즉시 기록되고 시스템 배너 옵션을 직접 결정합니다 (`AppDelegate.swift:60`).

## 4. 사용자가 그룹을 변경한다 (구독/해지 + 롤백)

설정 화면에서 청년부 구독을 해지하는 시나리오.

```mermaid
sequenceDiagram
    autonumber
    participant U as 사용자
    participant View as MoreView
    participant GS as GroupStore
    participant Sync as GroupSync.apply
    participant PS as PushService
    participant FCM as FCM
    participant API as BKCAPIClient
    participant WP as WordPress

    U->>View: "청년부" 토글 OFF
    View->>GS: setGroups(["all"])
    GS->>GS: 'all' 포함 확인 → 통과
    Note over GS: cannotUnsubscribeAll 가드 (IRON RULE #3)
    GS->>GS: toAdd=[], toRemove=["youth"]
    GS->>Sync: apply(toAdd, toRemove, ops)
    Sync->>PS: unsubscribe(fromGroup: "youth")
    PS->>FCM: unsubscribe topic 'bkc_youth'

    alt FCM 성공
        FCM-->>PS: ok
        Sync-->>GS: 성공
        GS->>API: updateGroups(deviceID, ["all"])
        API->>WP: PUT /subscribe/groups
        WP-->>API: 200
        GS->>GS: UserDefaults 갱신
    else FCM 오류
        FCM-->>PS: error
        Sync->>Sync: 이미 성공한 subscribe들 reverse rollback
        Sync-->>GS: throw partialFailure(error)
        GS-->>View: error 표시 ("재시도 해주세요")
    end
```

여러 그룹 동시 변경 시 (예: 청년부 OFF + 새가족 ON), 도중 실패 시 모든 부분 변경이 자동 롤백됩니다 ([IRON RULE #5](../doc/testing.md)).

## 5. 통계 집계 (Stats Rollup)

매시간 누적된 raw event들을 캠페인별 집계 카운터로 변환.

```mermaid
sequenceDiagram
    autonumber
    participant AS as Action Scheduler<br/>(매시간 cron)
    participant Roll as BKC_Stats_Rollup
    participant DB as MySQL
    participant Events as wp_bkc_campaign_events
    participant Stats as wp_bkc_campaign_stats
    participant Prune as BKC_Events::prune_older_than
    participant WPOpt as wp_options

    AS->>Roll: rollup_handler()
    Roll->>DB: SELECT DISTINCT campaign_uuid<br/>WHERE last_rolled_up IS NULL OR server_received_at > last_rolled_up
    DB-->>Roll: [uuid1, uuid2, uuid3]
    loop 각 캠페인
        Roll->>Events: SELECT event_type, COUNT(DISTINCT device_id) GROUP BY event_type
        Events-->>Roll: [{delivered:412}, {opened:201}, {deeplinked:88}]
        Roll->>Stats: UPSERT (campaign_uuid, delivered, opened, deeplinked, last_rolled_up=NOW())
        Note over Roll,Stats: ON DUPLICATE KEY UPDATE — 매번 덮어쓰기<br/>(IRON RULE #7: 멱등 — 두 번 돌려도 같은 숫자)
    end
    Roll->>WPOpt: get_option('bkc_last_event_prune')
    alt 마지막 prune 후 1일 경과
        Roll->>Prune: prune_older_than(6 months)
        Prune->>Events: DELETE WHERE server_received_at < NOW()-INTERVAL 6 MONTH
        Roll->>WPOpt: update_option('bkc_last_event_prune', time())
    end
```

핵심: `rollup_single`은 **누적 INCREMENT가 아니라 COUNT DISTINCT 재계산**입니다. 그래서 어떤 시점에 잘못 돌아도 다음 실행 때 자동으로 정정됩니다.

## 6. Universal Link로 외부 사이트에서 앱 열기 (보너스)

사용자가 카톡/메일에서 `https://bkc.org/sermon/2024-12-25` 링크를 탭한 경우.

```mermaid
sequenceDiagram
    autonumber
    participant U as 사용자
    participant Ext as 외부 앱 (카톡 등)
    participant iOS as iOS 시스템
    participant AASA as bkc.org/.well-known/<br/>apple-app-site-association
    participant App as BKC App
    participant ADel as AppDelegate
    participant DLR as DeepLinkRouter

    Note over iOS,AASA: 앱 설치 시 1회 fetch + 캐싱
    U->>Ext: 링크 탭
    Ext->>iOS: openURL(https://bkc.org/sermon/...)
    iOS->>iOS: AASA 매칭 → /sermon/* 패턴 hit
    iOS->>App: launch + NSUserActivity (browsingWeb)
    App->>ADel: continue userActivity
    ADel->>DLR: route(url)
    DLR->>DLR: DeepLink.from(url) → .sermon
    DLR->>App: selectedTab = 2 (설교 탭)
    App->>U: 설교 탭 표시
```

**전제:** `well-known/apple-app-site-association` 파일이 `https://bkc.org/.well-known/apple-app-site-association`로 정확하게 서비스 중이어야 함. Content-Type은 `application/json`. 리다이렉트 금지. CDN/WAF에서 Apple AASA bot을 화이트리스트해야 함.
